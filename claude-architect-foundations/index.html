1 About the certification

The Claude Certified Architect — Foundations credential proves you can design and build production AI systems on Anthropic's platform: integrating Claude via the API, engineering reliable prompts, wiring up tools and retrieval, extending Claude through the Model Context Protocol (MCP) and Skills, and orchestrating multi-step agents.

Who it's for

  • Technical architects designing scalable, AI-integrated systems on Claude.
  • Backend / full-stack engineers embedding Claude into apps and services.
  • Partner solution engineers who need a verifiable credential for customer engagements.

What the credential signals

  • You can authenticate and call the Claude API correctly, including streaming and structured output.
  • You can engineer, evaluate, and cache prompts for quality and cost control.
  • You can implement tool use, RAG/agentic search, and MCP servers/clients.
  • You can choose between workflows and agents, and architect parallelization, chaining, and routing.

5 Building with the Claude API Core

The fundamentals of talking to Claude programmatically: requests, conversations, system prompts, and output control.

The Messages API in one request

from anthropic import Anthropic
client = Anthropic()  # reads ANTHROPIC_API_KEY

resp = client.messages.create(
    model="claude-sonnet-4-5",
    max_tokens=1024,
    system="You are a concise technical assistant.",
    messages=[
        {"role": "user", "content": "Explain idempotency in one sentence."}
    ],
)
print(resp.content[0].text)

Core concepts the quiz tests

ConceptWhat to know
messagesAn alternating list of user / assistant turns. You resend the whole history each call — the API is stateless.
systemTop-level parameter (not a message) that sets role, rules, and tone. Stable system prompts are ideal cache targets.
max_tokensCaps the output length. Required.
temperature0 = deterministic/factual; higher = more varied/creative. Lower it for evals, extraction, and tool use.
Streamingstream=True yields tokens incrementally for responsive UIs.
Structured outputAsk for JSON and constrain with examples + XML tags, or use tool use to force a schema.
Stop reasonCheck resp.stop_reason (end_turn, max_tokens, tool_use, stop_sequence).
Multi-turn patternAppend the model's assistant reply to your messages list, then add the next user turn. Keep the array ordered and alternating.

6 Prompt engineering & evaluation Core

Reliable systems depend on reliable prompts — and on measuring them.

Prompt engineering techniques

  • Be clear and direct. State the task, constraints, and output format explicitly. Don't imply.
  • Be specific. Replace vague adjectives with concrete criteria ("3 bullets, ≤12 words each").
  • Structure with XML tags. Wrap inputs and sections (<document>, <example>, <instructions>) so Claude can parse roles unambiguously.
  • Provide examples (multishot). A few high-quality input→output pairs beat long prose instructions.
  • Give Claude room to think with extended thinking for hard reasoning tasks.
<instructions>
Classify the support ticket. Respond ONLY with JSON: {"category": ..., "urgency": ...}.
</instructions>
<example>
  <ticket>My card was charged twice</ticket>
  <output>{"category":"billing","urgency":"high"}</output>
</example>
<ticket>How do I reset my password?</ticket>

The evaluation workflow

An eval is how you prove a prompt works before and after changes. The course teaches this loop:

StepWhat you do
1. Build a test datasetGenerate representative inputs (Claude can help author them) with expected outcomes.
2. Run the promptExecute against every test case, capturing outputs.
3. GradeCode-based for exact/structural checks; model-based (Claude as judge) for open-ended quality.
4. IterateChange one thing, re-run, compare scores. Keep the dataset fixed so results are comparable.
Exam trapModel-based grading is for subjective quality (tone, helpfulness, correctness of free text). Code-based grading is for deterministic checks (valid JSON, contains keyword, exact match). Pick the cheaper deterministic option when the answer is checkable.

7 Tool use, features & RAG Core

Extending Claude beyond text: calling functions, searching knowledge, and using built-in capabilities.

Tool use (function calling)

  1. You define tool schemas (name, description, JSON input schema) and pass them with the request.
  2. Claude decides whether to call a tool and returns a tool_use block with arguments.
  3. Your code runs the real function, then returns a tool_result block referencing the tool_use_id.
  4. Claude continues the conversation using the result. Repeat for multi-step / multi-tool work.
tools = [{
  "name": "get_weather",
  "description": "Get current weather for a city.",
  "input_schema": {
    "type": "object",
    "properties": {"city": {"type": "string"}},
    "required": ["city"]
  }
}]

resp = client.messages.create(
    model="claude-sonnet-4-5", max_tokens=512,
    tools=tools,
    messages=[{"role":"user","content":"What's the weather in Amsterdam?"}],
)
# resp.stop_reason == "tool_use"  -> run the function, send tool_result back
RememberThe schema's description fields are prompt engineering — clear descriptions make Claude pick and fill tools correctly. Good tools, web search, the text-edit tool, and fine-grained / parallel tool calling are all tested.

RAG & agentic search

PieceWhy it matters
ChunkingSplit documents into retrievable units. Too big = noise; too small = lost context.
EmbeddingsVector representations enabling semantic similarity search.
BM25Lexical/keyword search — strong for exact terms, IDs, code. Complements embeddings.
Multi-index / hybridCombine semantic + BM25 (and re-rank) for best recall and precision.
Contextual retrievalPrepend chunk-level context before embedding to reduce ambiguity.
The RAG flowQuery → retrieve top-k → inject into prompt → Claude answers grounded in sources.

Features of Claude

  • Extended thinking — visible reasoning budget for complex problems.
  • Image & PDF support — multimodal inputs for analysis and extraction.
  • Citations — ground answers in supplied sources with verifiable references.
  • Prompt caching — cache stable prefixes (system prompt, docs, tools) to cut latency and cost. Know the rules: cache the static prefix, keep dynamic content after it.
  • Code execution & the Files API — run code and handle file inputs/outputs.

8 Model Context Protocol (MCP) Core

MCP is the open standard for connecting Claude to external tools and data without rewriting integrations for every app.

The big idea

MCP shifts the burden of defining and executing tools off your app and onto specialized, reusable MCP servers. It's transport-agnostic: a client and server exchange typed messages over any transport.

The three primitives

PrimitiveControlled byUse for
ToolsModelActions Claude can choose to invoke (read/write, call an API).
ResourcesApplicationRead-only data exposed via URIs (static or templated with parameters).
PromptsUserPre-crafted, high-quality instruction templates for common workflows.

Building a server (Python SDK)

from mcp.server.fastmcp import FastMCP
mcp = FastMCP("docs")

@mcp.tool()
def read_doc(doc_id: str) -> str:
    """Return the contents of a document."""
    return DOCS[doc_id]

@mcp.resource("docs://{doc_id}")
def doc_resource(doc_id: str) -> str:
    return DOCS[doc_id]
  • Decorators (@mcp.tool()) replace hand-written JSON schemas — type hints + docstrings become the schema.
  • Use the MCP Inspector (browser UI) to test tools, resources, and prompts.
  • Clients handle the request→response flow, read resources with correct MIME types, and surface prompts to users.
Exam trapMatch the primitive to its controller: tools = model-controlled, resources = app-controlled, prompts = user-controlled. This mapping is a favourite question.

9 Agent Skills Core

Skills are reusable markdown instructions Claude auto-applies to the right task at the right time. Teach once, reuse everywhere.

Anatomy of a Skill

---
name: pdf-report
description: Generate a branded PDF report from a data table.
                 Triggers when the user asks for a PDF/report deliverable.
allowed-tools: [shell, filesystem]
---
# Instructions
1. Read the input table...
2. Render with the template in ./assets/template.html ...
  • SKILL.md frontmatter — name and a precise description drive trigger matching. Vague descriptions = unreliable triggering.
  • Progressive disclosure — keep the main file lean; push detail into linked files/scripts so the context window stays efficient.
  • allowed-tools — restrict what a Skill may do.
  • Scripts execute without consuming context — offload deterministic work.

Skills vs other customization

MechanismBest for
SkillsReusable, auto-triggered task expertise.
CLAUDE.mdAlways-on project memory/standards.
HooksDeterministic control (formatting, blocking commands, notifications).
SubagentsIsolated, delegated expert tasks with their own context.

Share Skills via a repo, distribute through plugins, or deploy org-wide with enterprise managed settings. Know the troubleshooting basics: skill won't trigger (fix the description), priority conflicts, runtime errors.

10 Agents & workflows Architect core

The architect's headline decision: when to use a structured workflow vs a self-directing agent.

Workflows vs agents

WorkflowAgent
Control flowYou define fixed, predictable steps.Claude decides the next step dynamically.
Best whenTask is well-understood and decomposable.Path is open-ended; needs tools + environment feedback.
Trade-offPredictable, cheaper, easier to test.Flexible, but costlier and harder to constrain.
Architect's ruleStart with the simplest thing that works. Use a workflow unless the task genuinely needs an agent's autonomy. Add complexity only when it earns its keep.

The three workflow patterns

  • Parallelization — fan out independent subtasks at once, then aggregate (e.g., multiple analyses, voting).
  • Chaining — pipe one step's output into the next for staged refinement.
  • Routing — classify the input, then dispatch to the specialized handler/prompt.

Agents combine an LLM core with tools and environment inspection (acting, observing results, deciding again) until a stop condition is met.

12 Flashcards & quiz prep

Click a card to reveal the answer. These mirror the kinds of questions in the per-module quizzes and final assessment.

Is the Messages API stateful or stateless?reveal
Stateless. You resend the entire conversation history (the ordered, alternating messages array) on every request. The API does not remember prior calls.
Where does the system prompt go, and what's it for?reveal
It's a top-level system parameter, not a message. It sets role, rules, and tone, and is an ideal prompt-cache target because it's stable across turns.
When do you choose code-based vs model-based grading?reveal
Code-based for deterministic, checkable outputs (valid JSON, exact match, contains keyword). Model-based (Claude as judge) for subjective quality like tone, helpfulness, or correctness of free-form text.
Name the four prompt-engineering techniques the course emphasizes.reveal
Be clear and direct; be specific; structure with XML tags; provide examples (multishot). Plus: give Claude room to think for hard reasoning.
Walk the tool-use loop.reveal
Send tools + message → Claude returns a tool_use block (stop_reason="tool_use") → you execute the function → return a tool_result with the matching tool_use_id → Claude continues. Repeat for multi-step/multi-tool.
Embeddings vs BM25 — when does each win?reveal
Embeddings capture semantic similarity (paraphrases, concepts). BM25 is lexical/keyword search, strong on exact terms, IDs, and code. Hybrid (both, then re-rank) gives the best retrieval.
Map the three MCP primitives to their controllers.reveal
Tools = model-controlled (Claude invokes). Resources = application-controlled (read-only data via URIs). Prompts = user-controlled (templated instructions the user picks).
What problem does MCP solve for an architect?reveal
It standardizes how Claude connects to external tools/data, shifting tool definition and execution onto reusable, transport-agnostic MCP servers instead of bespoke per-app integrations.
What makes a Skill trigger reliably?reveal
A precise description in the SKILL.md frontmatter that clearly states when to use it. Vague descriptions are the #1 cause of skills not triggering.
Skill vs CLAUDE.md vs hook vs subagent?reveal
Skill = reusable auto-triggered expertise. CLAUDE.md = always-on project memory. Hook = deterministic control (formatting, blocking, notifications). Subagent = isolated delegated task with its own context.
Workflow or agent — and why start simple?reveal
Workflow = you define fixed steps (predictable, cheaper, testable). Agent = Claude self-directs with tools + feedback (flexible, costlier). Start with the simplest approach; add agent autonomy only when the task truly needs it.
Name the three workflow patterns.reveal
Parallelization (fan out independent subtasks, aggregate), Chaining (pipe output to next step), Routing (classify then dispatch to a specialized handler).
What is prompt caching and what do you cache?reveal
A cost/latency optimization that reuses a previously processed static prefix. Cache the stable parts (system prompt, large docs, tool definitions) and keep dynamic content after the cached prefix.
How does temperature affect output, and where do you set it low?reveal
Higher temperature = more varied/creative; 0 = deterministic. Use low/zero for evals, extraction, classification, and tool use where consistency matters.

13 Final assessment strategy

  • Pass the module quizzes first. The final assessment recombines them — if you can pass each quiz cleanly, the final is mostly recall.
  • Master the mapping questions. MCP primitives ↔ controllers, workflow patterns, grading types, and Skill-vs-alternatives are high-frequency.
  • Know the loops cold. Be able to recite the tool-use loop and the eval workflow without notes.
  • Default to "simplest that works." For scenario questions, the right architecture is usually the least complex one that meets the requirement.
  • Do the hands-on exercises. Building one real MCP server and one Skill cements the concepts the multiple-choice can't fully test.
  • Re-watch at 1.5x any lesson whose quiz question you missed, then retake.
Day-of tipsRead each scenario twice. Eliminate obviously-too-complex options. Watch for "model-controlled vs app-controlled" and "workflow vs agent" wording — those distinctions are where points are won or lost.
